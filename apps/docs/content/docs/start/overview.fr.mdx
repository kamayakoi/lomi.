lomi. est une plateforme de paiement pour les entreprises qui vendent en Afrique de l’Ouest francophone. Elle vous permet d’accepter les cartes et le mobile money, d’envoyer des liens de paiement, d’utiliser un checkout hébergé, de gérer les clients et abonnements, de recevoir des webhooks et de suivre les transactions et les reversements.

## Quand utiliser lomi. ?

Utilisez lomi. si vous voulez :

- Accepter Wave, MTN MoMo, Orange Money, SPI, les cartes et d’autres moyens de paiement locaux avec une seule intégration.
- Rediriger vos clients vers un checkout hébergé au lieu de construire toute la collecte de paiement.
- Créer des liens de paiement pour des factures, services, événements, dons ou produits.
- Gérer des paiements récurrents et des abonnements clients.
- Recevoir des événements webhook fiables pour les paiements, remboursements, abonnements et états de checkout.
- Suivre soldes, transactions, remboursements et reversements depuis le tableau de bord.

## Complexité des paiements

Le paysage des paiements en Afrique de l’Ouest est fragmenté entre de nombreux acteurs. Les marchands gèrent souvent plusieurs intégrations ou dépendent d’acquéreurs propriétaires, ce qui augmente délais, coûts et dette technique.

lomi. y répond avec :

- **APIs** : une intégration pour cartes, MTN MoMo, Wave, Orange Money, SPI (64+ opérateurs mobile money et banques), et plus.
- **Tableau de bord** : back-office web pour vendre biens, services, abonnements, etc.
- **Application mobile** : soft-POS pour vendre en mobilité et retirer vos revenus.
- **Contrôle** : rapprochement, remboursements, reversements, checkout hébergé, liens de paiement, éléments intégrables, API de charges directes et rapports.

> **Commencez sur l’Afrique de l’Ouest francophone en [créant un compte](https://lomi.africa/sign-up).**

## Transparence et conformité

lomi. vise la visibilité sur les frais, les flux de transaction et les reversements :

- **Open source** : la majeure partie du code est public — voir la [structure du code](/resources/open-source/codebase).
- **Gouvernance communautaire** : feuille de route et retours via issues et pull requests GitHub.
- **Conformité** : agrément BCEAO (en cours), PCI DSS niveau 4, partenaires carte certifiés, RGPD et MACP avec flux hébergés notamment au Cap et à Dublin.

Pour la vision produit complète, voir le [manifeste](/resources/manifesto).

## Comment la plateforme s’assemble ?

Le parcours habituel est simple :

1. Créez un compte et terminez l’onboarding.
2. Récupérez vos clés API de test dans le tableau de bord (ou `lomi login` pour la CLI).
3. Créez une session de checkout avec `lomi checkout create`, un lien de paiement ou une demande de paiement.
4. Laissez le client payer avec un moyen disponible.
5. Confirmez le résultat via l’API, le tableau de bord ou un webhook.
6. Passez aux clés live quand l’intégration est prête.

La référence API donne les détails exacts des requêtes et réponses. Les guides Build expliquent quel produit utiliser et comment livrer le flux.
