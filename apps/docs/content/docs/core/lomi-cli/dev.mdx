---
title: 'Local development server'
description: 'Run lomi dev to receive and inspect webhook events on your machine during development.'
---

The `lomi dev` command starts a lightweight local HTTP server that receives webhook POST requests and prints events in the terminal. Use it while building and debugging webhook handlers.

Requires [`lomi login`](/core/lomi-cli/auth).

## Usage

```bash filename="Terminal"
lomi dev
```

Default URL: **`http://localhost:4242/webhook`**

### Options

| Flag | Default | Description |
| --- | --- | --- |
| `-p, --port` | `4242` | Port for the HTTP server |
| `--env-file` | `.env` | Dotenv file to load |
| `--verify-signature` | `false` | Verify `lomi-signature` using `LOMI_WEBHOOK_SECRET` |
| `--skip-rules-install` | — | Skip agent rules install prompt |

```bash filename="Terminal"
lomi dev --port 3000
lomi dev --verify-signature
```

## Endpoints

| Method | Path | Description |
| --- | --- | --- |
| `POST` | `/webhook` | Receives webhook event payloads |
| `GET` | `/health` | Returns `ok` |

## Testing locally

<Steps>

### Start the dev server

```bash filename="Terminal"
lomi dev
```

### Send a test event

```bash filename="Terminal"
curl -X POST http://localhost:4242/webhook \
  -H "Content-Type: application/json" \
  -d '{"type":"test.event","data":{}}'
```

The CLI prints the event type and payload in the terminal.

### Connect to the dashboard (optional)

For end-to-end testing, expose your local server with a tunnel (e.g. [ngrok](https://ngrok.com)) and register the public URL as a webhook endpoint in the Merchant Portal.

</Steps>

## Signature verification

When `--verify-signature` is enabled, the server validates the `lomi-signature` header against `LOMI_WEBHOOK_SECRET` from your `.env` file.

Ensure `LOMI_WEBHOOK_SECRET` is set before enabling verification:

```dotenv filename=".env"
LOMI_WEBHOOK_SECRET=whsec_your_secret_here
```

For production webhook handling patterns, see [Handling webhooks](/core/advanced-guides/handling-webhooks) and [Webhooks reference](/reference/payments/webhooks).

## Relationship to `lomi init`

Running `lomi dev` after [`lomi init`](/core/lomi-cli/init) loads credentials from the project `.env`. The generated `examples/webhook-handler.ts` shows how to implement verification in your own application code.
