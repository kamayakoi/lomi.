La CLI lomi. utilise deux niveaux de configuration : **global** (authentification) et **projet** (clés SDK).

## Config globale CLI

Après [`lomi login`](/core/lomi-cli/auth) :

| Plateforme | Chemin |
| --- | --- |
| macOS | `~/Library/Application Support/lomi/config.json` |
| Linux | `~/.config/lomi/config.json` |

Surcharge : `export LOMI_CONFIG_DIR=/chemin/custom/lomi`

## Variables d’environnement CLI

| Variable | Description |
| --- | --- |
| `LOMI_ACCESS_TOKEN` | Jeton CLI pour CI (prioritaire sur le profil) |
| `LOMI_CONFIG_DIR` | Répertoire de config global |
| `LOMI_SUPABASE_ANON_KEY` | Clé anon Supabase pour device auth (optionnel) |

## Variables projet (`.env`)

Créées par [`lomi init`](/core/lomi-cli/init) :

| Variable | Description |
| --- | --- |
| `LOMI_API_KEY` | Clé secrète SDK |
| `LOMI_WEBHOOK_SECRET` | Secret de signature webhook |
| `LOMI_API_URL` | URL de base API |

<Callout type="warning" emoji="⚠️" title="Deux types d’identifiants">
  **`LOMI_ACCESS_TOKEN`** — authentifie la **CLI**.

  **`LOMI_API_KEY`** — authentifie le **SDK** dans votre application.
</Callout>

## Options globales

```bash filename="Terminal"
lomi --profile sandbox status
lomi -a https://sandbox.api.lomi.africa checkout create
```

## URLs API

| Environnement | URL |
| --- | --- |
| Production | `https://api.lomi.africa` |
| Sandbox | `https://sandbox.api.lomi.africa` |

## Exemple CI

```bash filename="Terminal"
export LOMI_ACCESS_TOKEN="$CLI_TOKEN"
export LOMI_API_KEY="$API_KEY"
lomi status
lomi init --yes -e sandbox -L ts --api-key "$LOMI_API_KEY" --skip-package-install
```
