## Initialiser un dépôt Git

À la racine du code et de la configuration :

```bash filename="Terminal"
git init
```

## Index et commits

Après modification des fichiers, indexez avec `git add` :

```bash filename="Terminal"
git add .
```

Puis commitez avec un message clair :

```bash filename="Terminal"
git commit -m "Update payment provider configuration"
```

## Pousser vers le dépôt distant

Pour partager vos changements :

```bash filename="Terminal"
git push origin develop
```

Remplacez par la bonne branche si votre stratégie diffère.

## Récupérer les changements distants

```bash filename="Terminal"
git pull origin develop
```

Adapter le nom de branche au besoin.

## Résoudre les conflits de fusion

En cas de conflit après `pull`, éditez les fichiers, puis :

```bash filename="Terminal"
git add resolved_file.js
git commit -m "Resolve merge conflicts"
```

## Bonnes pratiques

- Commits fréquents et descriptifs
- Branches dédiées aux fonctionnalités ou configurations
- `pull` régulier depuis le dépôt distant
- Résolution de conflits soigneuse et communication d’équipe

Une utilisation disciplinée de Git fluidifie la collaboration sur votre projet lomi..
