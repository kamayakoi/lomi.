Nous proposons à la fois une offre open source et une plateforme hébergée pour répondre aux besoins variés des marchands. Dans un souci de transparence et de collaboration communautaire, nous appliquons une **double licence**.

Le projet est principalement sous **GNU Affero General Public License (AGPL)** : la communauté peut utiliser, modifier et redistribuer le logiciel librement. L’AGPL impose toutefois que toute utilisation en service (SaaS notamment) **publie également le code source**. Si vous souhaitez utiliser **lomi.** sans publier vos modifications (intégration à un produit propriétaire) ou l’exploiter commercialement pour des offres similaires, **une licence commerciale est requise**.

Certains composants relèvent exclusivement de notre **licence commerciale**, pour les entreprises ayant besoin de fonctionnalités additionnelles ou d’une plus grande flexibilité. Ce **double modèle** nous permet de servir à la fois la communauté open source et les partenaires enterprise.

Voici un aperçu des options de déploiement et comment démarrer.

## Auto-hébergement

Pour héberger la plateforme vous-même sans payer nos frais technologiques, clonez le dépôt principal sur GitHub : vous accédez à la dernière version et pouvez l’adapter. Vous devrez toutefois obtenir directement auprès des prestataires de paiement les accords commerciaux et clés API nécessaires au plein usage en libre-service.

Pour l’organisation du code, voir la page [Codebase](/core/freedom/codebase).

Les marchands qui veulent simplement accepter des paiements peuvent utiliser lomi. sur notre site sans contrainte majeure ni configuration spéciale. L’auto-hébergement est possible mais rarement recommandé pour la plupart des marchands : vous perdez alors les avantages du **Merchant of Record** dédié ainsi que la rapidité et la simplicité que lomi. apporte à votre activité.

## Documentation et auto-hébergement

Documentation complète et informations d’auto-hébergement : [github.com/lomiafrica/lomi.](https://github.com/lomiafrica/lomi./).

### Démarrage rapide

```bash copy filename="Terminal"
# Clone the repository
git clone https://github.com/lomi-africa/lomi./
cd lomi.

# Install dependencies
bun install

# Set up environment
cp .env.example .env

# Set up database
supabase link

# Apply database migrations
supabase db push

# Start development server
bun run dev
```
