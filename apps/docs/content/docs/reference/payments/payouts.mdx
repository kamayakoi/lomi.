Merchant-initiated payouts on supported rails are exposed as **`POST`** routes with provider-specific payloads. There is currently **no** generic `GET /payouts` list API in the public merchant contract—track completion with **webhooks**.

<Callout type="warning">
  Payout initiation is asynchronous. Listen for webhook events (`payout.completed`, `payout.failed`, …) rather than polling a list endpoint that does not exist on the public REST surface yet.
</Callout>

Lifecycle, limits, fees: **[Payout lifecycle](/reference/payments/payout-lifecycle)**.

## Initiate a Wave payout

Maps to `POST /payout/wave`. Request payload matches the Nest DTO (`amount`, `currency`, nested `beneficiary`).

<Tabs items={["TypeScript", "Python", "cURL"]}>

<Tab value="TypeScript">
```typescript
import { LomiSDK } from '@lomi./sdk';

const lomi = new LomiSDK({
  apiKey: process.env.LOMI_API_KEY!,
  environment: 'live',
});

await lomi.payouts.createWavePayout({
  amount: 25000,
  currency: 'XOF',
  beneficiary: {
    name: 'Aicha Diallo',
    phoneNumber: '+221771234567',
  },
  reason: 'Weekly settlement',
});
```
</Tab>

<Tab value="Python">
```python
# Mirrors the Wave payout edge contract (field names mirror server expectation)
response = requests.post(
    "https://api.lomi.africa/payout/wave",
    headers={"X-API-KEY": os.environ["LOMI_API_KEY"], "Content-Type": "application/json"},
    json={
        "amount": 25000,
        "currency": "XOF",
        "beneficiary": {"name": "Aicha Diallo", "phoneNumber": "+221771234567"},
        "reason": "Weekly settlement",
    },
)
print(response.status_code, response.json())
```
</Tab>

<Tab value="cURL">
```bash
curl -X POST "https://api.lomi.africa/payout/wave" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 25000,
    "currency": "XOF",
    "beneficiary": { "name": "Aicha Diallo", "phoneNumber": "+221771234567" },
    "reason": "Weekly settlement"
  }'
```
</Tab>

## Field reference (Wave payout)

| Field | Type | Required | Description |
| --- | --- | --- | --- |
| `amount` | `number` | **Yes** | Amount to send (minimums enforced server-side). |
| `currency` | `string` | **Yes** | ISO currency (`XOF`, …). |
| `beneficiary` | `object` | **Yes** | Recipient `name` + `phoneNumber` (mobile-money ID). |
| `reason` | `string` | No | Ledger / audit memo. |
