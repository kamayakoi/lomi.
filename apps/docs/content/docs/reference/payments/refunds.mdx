Public merchant refunds are exposed as **`POST /refund/wave`** with `{ transactionId, amount, reason? }`. There is **no** generic refunds list or single-resource `GET` on the public REST surface today—use **webhooks** and transaction history for reconciliation.

<Callout type="warning">
  Match `transactionId` carefully; refunds are asynchronous and provider responses stream through webhooks.
</Callout>

## Create a refund

<Tabs items={["TypeScript", "Python", "cURL"]}>

<Tab value="TypeScript">
```typescript
import { LomiSDK } from '@lomi./sdk';

const lomi = new LomiSDK({
  apiKey: process.env.LOMI_API_KEY!,
  environment: 'live',
});

await lomi.refunds.createWaveRefund({
  transactionId: '123e4567-e89b-12d3-a456-426614174000',
  amount: 5000,
  reason: 'duplicate_charge',
});
```
</Tab>

<Tab value="Python">
```python
import requests, os

r = requests.post(
    "https://api.lomi.africa/refund/wave",
    headers={"X-API-KEY": os.environ["LOMI_API_KEY"], "Content-Type": "application/json"},
    json={
        "transactionId": "123e4567-e89b-12d3-a456-426614174000",
        "amount": 5000,
        "reason": "duplicate_charge",
    },
)
print(r.status_code, r.json())
```
</Tab>

<Tab value="cURL">
```bash
curl -X POST "https://api.lomi.africa/refund/wave" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "transactionId": "123e4567-e89b-12d3-a456-426614174000",
    "amount": 5000,
    "reason": "duplicate_charge"
  }'
```
</Tab>

</Tabs>

## Payload reference

| Field | Type | Required |
| --- | --- | --- |
| `transactionId` | `string` (UUID) | **Yes** |
| `amount` | `number` | **Yes** |
| `reason` | `string` | No |
