The Payment Intents API is used for **card-form-only** integrations where your app renders card fields (for example with `@lomi./sdk`) and confirms payment client-side.

<Callout type="info">
  Use this API when you need an embedded card form.  
  If you prefer a hosted page (redirect or iframe), use [Checkout Sessions](/reference/payments/checkout-sessions).
</Callout>

## Create a payment intent

`POST /payment-intents`

### Authentication

You can authenticate with either:

- `X-API-KEY: <your-key>`
- `Authorization: Bearer <your-key>`

### Request body

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `amount` | `number` | **Yes** | Amount to charge in the original currency |
| `currency_code` | `string` | No | `XOF`, `USD`, `EUR` (defaults to `XOF`) |
| `currency` | `string` | No | Backward-compatible alias for `currency_code` |
| `customer_id` | `string` (UUID v4) | **One path required**¹ | Internal customer ID |
| `customer_email` | `string` | **One path required**¹ | Valid email when using guest path |
| `customer_name` | `string` | **One path required**¹ | Display name when using guest path |
| `customer_phone` | `string` | No | Customer phone |
| `description` | `string` | No | Payment description |
| `payment_reference` | `string` | No | Merchant reference (invoice, order ID, etc.) |
| `product_id` | `string` (UUID v4) | No | Optional product UUID for metadata |
| `subscription_id` | `string` (UUID v4) | No | Optional subscription UUID for metadata |
| `quantity` | `number` | No | Quantity hint for reconciliation |
| `metadata` | `object` | No | Custom metadata |

¹ Send **`customer_id`** **or** **`customer_email` + `customer_name`** so the API can create/find an internal customer before charging. Otherwise the request returns **400**.

### Response

```json
{
  "success": true,
  "data": {
    "id": "pi_3QxYk6...",
    "client_secret": "pi_3QxYk6..._secret_...",
    "amount": 152,
    "currency": "eur",
    "original_amount": 10000,
    "original_currency": "XOF",
    "status": "requires_payment_method"
  }
}
```

## End-to-end example

### 1) Server: create payment intent

<Tabs items={['Node', 'cURL']}>
  <Tab value="Node">
    ```typescript
    const response = await fetch('https://api.lomi.africa/payment-intents', {
      method: 'POST',
      headers: {
        'X-API-KEY': process.env.LOMI_API_KEY!,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        amount: 10000,
        currency_code: 'XOF',
        customer_email: 'customer@example.com',
        customer_name: 'Ada Lovelace',
        description: 'Order #ORD-12345',
        payment_reference: 'ORD-12345',
      }),
    });

    const body = await response.json();
    const clientSecret = body.data.client_secret;
    ```
  </Tab>

  <Tab value="cURL">
    ```bash
    curl -X POST "https://api.lomi.africa/payment-intents" \
      -H "X-API-KEY: $LOMI_API_KEY" \
      -H "Content-Type: application/json" \
      -d '{
        "amount": 10000,
        "currency_code": "XOF",
        "customer_email": "customer@example.com",
        "customer_name": "Ada Lovelace",
        "description": "Order #ORD-12345"
      }'
    ```
  </Tab>
</Tabs>

### 2) Client: mount card form and confirm

```typescript
import { loadLomi } from '@lomi./sdk';

const lomi = await loadLomi('lomi_pk_your_publishable_key');
const elements = lomi.elements({ clientSecret });
const paymentElement = elements.create('payment');
paymentElement.mount('#payment-element');

const { error } = await lomi.confirmPayment({
  elements,
  confirmParams: {
    return_url: 'https://example.com/payments/success',
  },
});

if (error) {
  console.error(error.message);
}
```

## Webhooks and reconciliation

- Subscribe to merchant webhook events like `PAYMENT_SUCCEEDED` and `PAYMENT_FAILED`.
- For reconciliation, provide **`customer_id`** or **`customer_email` + `customer_name`**, plus merchant references (`payment_reference`, `metadata.order_id`) where possible.
- The API creates a pending internal transaction for every successful intent; failures to register that row return **400** (retry or fix data).
- Use [Webhooks](/reference/payments/webhooks) for signature verification and retry behavior.
