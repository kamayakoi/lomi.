## Overview

Create a **payment request** for one of your customers to pay a known amount by a **deadline**.

The request is trackable through its lifecycle (`pending` → `completed` / `failed` / `expired` via other flows and webhooks depending on your integration).

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`POST /payment-requests`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

_No query parameters._

### Request body

Required fields per OpenAPI and `CreatePaymentRequestDto` in `apps/api/src/core/payment-requests/dto/create-payment-request.dto.ts`:

| Field | Required | Notes |
| --- | --- | --- |
| `amount` | Yes | Amount to request (see your product’s smallest unit rules) |
| `currency_code` | Yes | One of `XOF`, `USD`, `EUR` |
| `expiry_date` | Yes | ISO-8601 datetime after which the request expires |
| `description` | Optional | Free text (invoice label, etc.) |
| `customer_id` | Optional | Existing customer UUID |
| `payment_reference` | Optional | Your invoice / order ref for reconciliation |
| `metadata` | Optional | Opaque object for your ops stack |

Example:

```json
{
  "amount": 25000,
  "currency_code": "XOF",
  "description": "Consulting — March",
  "expiry_date": "2026-12-31T23:59:59.000Z",
  "payment_reference": "INV-2026-042",
  "metadata": { "department": "finance" }
}
```

## Responses

**201** — body is `PaymentRequestResponseDto` (ids, status, timestamps as returned by the service).

| Status | Description |
| --- | --- |
| `201` | Demande de paiement créée avec succès |
| `400` | Entrée invalide ou erreur de validation |
| `401` | Clé API invalide ou manquante |

## Errors

**400** when `expiry_date` is not a valid datetime, `currency_code` outside allowed values, or numeric validation fails.

**401** when the API key is missing or not valid for the environment.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/payment-requests" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount":25000,
    "currency_code":"XOF",
    "expiry_date":"2026-12-31T23:59:59.000Z",
    "payment_reference":"INV-2026-042"
  }'
```
