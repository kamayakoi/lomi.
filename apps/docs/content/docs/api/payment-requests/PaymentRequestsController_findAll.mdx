## Overview

Paginated list of **payment requests** for your organization. Filter by status and/or customer id.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /payment-requests`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

| Name | Defaults | Notes |
| --- | --- | --- |
| `status` | (any) | One of `pending`, `completed`, `failed`, `expired` |
| `customerId` | (optional) | Narrow to a single customer UUID |
| `limit` | `20` | Page size |
| `offset` | `0` | Zero-based offset |

## Responses

**200** returns a **wrapper object** (not a bare array):

| Field | Meaning |
| --- | --- |
| `data` | `PaymentRequestResponseDto[]` |
| `total` | Total matching rows |
| `limit` / `offset` | Echo of paging params |

| Status | Description |
| --- | --- |
| `200` | Liste paginĂŠe de demandes |
| `401` | ClĂŠ API invalide ou manquante |

## Errors

**401** Unauthorized for bad key.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/payment-requests" \
  --data-urlencode "status=pending" \
  --data-urlencode "limit=20" \
  --data-urlencode "offset=0" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
