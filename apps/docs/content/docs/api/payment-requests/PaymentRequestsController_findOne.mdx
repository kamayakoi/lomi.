## Overview

Fetch a single payment request by UUID: current status and payment metadata as stored for your organization.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /payment-requests/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | Payment request UUID |

### Query parameters

_No query parameters._

## Responses

| Status | Description |
| --- | --- |
| `200` | Détails de la demande |
| `401` | Clé API invalide ou manquante |
| `404` | Demande introuvable ou accès refusé |

## Errors

**404** if the id does not exist or is not visible to your organization (no resource enumeration across tenants).

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/payment-requests/YOUR_REQUEST_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
