## Overview

**Inbound Stripe→lomi webhook receiver.** Stripe POSTs signed event payloads here; this route is **not** called with merchant `X-API-KEY` — Stripe authenticates via **`Stripe-Signature`**.

Implementation uses **raw bytes** (`rawBody`) for cryptographic verification — see `apps/api/src/webhooks/providers/stripe/stripe-webhook.controller.ts` and Express middleware wiring in `apps/api/src/main.ts`.

Configure this URL as the Stripe webhook endpoint in the Stripe Dashboard (test vs live URLs).

## Authentication

These requests are authenticated by Stripe using **`Stripe-Signature`** (your Stripe signing secret stays in lomi infra / dashboard—not in merchant API docs). **Do not** send `Authorization: Bearer` or merchant `X-API-KEY` from Stripe servers.

### Headers

| Header | Required | Notes |
| --- | --- | --- |
| `Stripe-Signature` | Yes | HMAC verification input |
| `Content-Type` | Typically `application/json` | Preserve body bytes |

## Endpoint

`POST /webhooks/stripe`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters._

### Query parameters

_None._

### Request body

The **exact JSON Stripe posts** (`event.json` structure). Middleware must expose the unchanged raw payload to Nest for signature verification; if `rawBody` is missing processing fails with HTTP error.

This is distinct from merchant JSON-RPC style APIs above.

## Responses

| Status | Description |
| --- | --- |
| `200` | Webhook reçu et traité avec succès — JSON includes `{ "received": true, ...processor fields }` |
| `400` | Signature webhook invalide or missing raw payload |
| `500` | Erreur interne lors du traitement du webhook |

## Errors

**400** malformed signature or middleware misconfiguration (**no merchant 401 here**). Stripe may retry deliveries on transient errors—implement idempotent processing on your Stripe integration side accordingly.

## Example

Merchant developers usually **never call this** manually. For diagnostics, simulate with Stripe CLI and forward preserving signature:

```bash
stripe trigger payment_intent.succeeded
# or forward an event Stripe CLI emits to your URL with proper Stripe-Signature
```

Manual curl placeholder (normally invalid without real signature):

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/webhooks/stripe" \
  -H "Content-Type: application/json" \
  -H "Stripe-Signature: $STRIPE_SIGNATURE" \
  --data-binary @stripe-event.json
```
