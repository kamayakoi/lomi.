## Overview

Retrieve a single webhook configuration by **UUID**.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /webhooks/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | Webhook UUID |

### Query parameters

_No query parameters._

## Responses

| Status | Description |
| --- | --- |
| `200` | Le webhook |
| `401` | Unauthorized |
| `404` | Not found for org |

## Errors

**404** if unknown id or not owned by this API key organization.

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/webhooks/YOUR_WEBHOOK_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
