## Overview

Partially updates a webhook endpoint configuration: delivery URL, active flag, event subscription filters, verification token, and metadata. Omit fields you do not want to change.

Implementation: `apps/api/src/webhooks/webhooks.controller.ts` — body matches `UpdateWebhookDto` in `apps/api/src/webhooks/dto/update-webhook.dto.ts`.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`PATCH /webhooks/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | Webhook UUID |

### Query parameters

_No query parameters._

### Request body

Partial JSON. Common fields:

| Field | Purpose |
| --- | --- |
| `url` | HTTPS delivery endpoint |
| `is_active` | Enable/disable delivery attempts |
| `authorized_events` | Encoded list / filter (per your org’s stored format) |
| `spi_event_types` | SPI-related subscriptions when supported |
| `supports_spi` | SPI toggle |
| `verification_token` | Secret your handler can echo for verification |
| `metadata` | Opaque object |

Example rotation:

```json
{
  "url": "https://example.com/webhooks/lomi",
  "is_active": true
}
```

## Responses

| Status | Description |
| --- | --- |
| `200` | Webhook mis à jour avec succès |
| `401` | Clé API invalide ou manquante |
| `404` | Resource not found for org |

## Errors

**401** bad key. **404** bad id or cross-tenant access.

## Example

```bash
curl -sS -X PATCH "https://sandbox.api.lomi.africa/webhooks/YOUR_WEBHOOK_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com/webhooks/lomi","is_active":true}'
```
