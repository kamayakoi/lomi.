## Overview

Lists **merchant-configured webhook endpoints** (URLs, subscribed events metadata, activation state) registered for your organization.

Use these records to correlate [webhook deliveries](/reference/payments/webhooks) and troubleshoot delivery failures.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /webhooks`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

_No query parameters._

## Responses

Returns an array of `WebhookResponseDto` (delivery URL, verification token, subscribed events serialization, etc.).

| Status | Description |
| --- | --- |
| `200` | Liste des webhooks |
| `401` | Unauthorized |

## Errors

**401:** missing/invalid merchant API key.

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/webhooks" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
