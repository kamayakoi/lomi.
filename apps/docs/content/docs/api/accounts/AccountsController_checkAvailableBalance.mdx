## Overview

Pre-flight helper for disbursement / automation: returns whether enough **available** balance exists right now for the given **`currency`** path segment (not query-based).

JSON fields (per Swagger schema):

- `has_sufficient_balance` (`boolean`)
- `available_balance` (`number`)

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /accounts/balance/check/{currency}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `currency` | Yes | `XOF`, `USD`, or `EUR` |

### Query parameters

_None._

## Responses

| Status | Description |
| --- | --- |
| `200` | Résultat de la vérification de solde |
| `401` | Clé API invalide ou manquante |

## Errors

**401** unauthorized.

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/accounts/balance/check/XOF" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
