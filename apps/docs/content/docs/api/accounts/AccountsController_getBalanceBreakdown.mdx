## Overview

Returns **available vs pending vs total** balance split (and optional FX conversion) for reporting and payout planning.

Use optional `target_currency` to express all rows in a unified reporting currency when your business needs consolidated views.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /accounts/balance/breakdown`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

| Name | Notes |
| --- | --- |
| `target_currency` | Optional FX target: `XOF`, `USD`, `EUR` |

## Responses

**200** — array of `BalanceBreakdownResponseDto`.

| Status | Description |
| --- | --- |
| `200` | Détail des soldes avec conversion |
| `401` | Clé API invalide ou manquante |

## Errors

**401** if key invalid.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/accounts/balance/breakdown" \
  --data-urlencode "target_currency=XOF" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
