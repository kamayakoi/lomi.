## Overview

Fetch a single account by UUID (metadata, currency context, status as returned by your org’s account model).

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /accounts/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | Account UUID |

### Query parameters

_None._

## Responses

| Status | Description |
| --- | --- |
| `200` | Le compte |
| `401` | Unauthorized |
| `404` | Compte introuvable |

## Errors

**404** when the UUID is unknown or belongs to another organization.

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/accounts/YOUR_ACCOUNT_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
