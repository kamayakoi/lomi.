La référence API documente l’API marchande publique : les endpoints qu’un développeur doit utiliser depuis une intégration serveur. Elle n’expose volontairement pas les services internes de providers, organisations, administration marchande ou agents.

Si vous choisissez encore votre chemin d’intégration, commencez par [Quelle intégration de paiement choisir ?](/build/choose-integration).

## URLs de base

| Environnement | URL de base | Clé secrète |
| --- | --- | --- |
| Test | `https://sandbox.api.lomi.africa` | `lomi_sk_test_...` |
| Live | `https://api.lomi.africa` | `lomi_sk_live_...` |

Envoyez la clé secrète dans l’en-tête `X-API-Key` depuis votre serveur.

## Conventions de référence

- Les guides expliquent les décisions produit et les flux complets.
- Les pages API décrivent les opérations HTTP exactes.
- Les encaissements directs sont avancés ; le checkout hébergé reste la recommandation par défaut.
- Utilisez la sandbox pour développer et le live seulement après la [checklist de production](/start/go-live).
