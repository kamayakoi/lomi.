## Overview

Initiates a **third-party payout** from your account balance (supplier, staff, vendor, etc.). The API returns immediately with a **pending** state; settlement is **asynchronous** on the provider (Wave, SPI). Status updates arrive through **your merchant webhooks** — see [Handling webhooks](/core/advanced-guides/handling-webhooks).

**400** covers validation errors and **insufficient balance** (per controller description).

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`POST /beneficiary-payouts`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

_None._

### Request body

`CreateBeneficiaryPayoutDto` (`apps/api/src/core/beneficiary-payouts/dto/create-beneficiary-payout.dto.ts`):

| Field | Required | Notes |
| --- | --- | --- |
| `amount` | Yes | Montant |
| `currency_code` | Yes | `XOF` \| `USD` \| `EUR` |
| `payout_method_id` | No | Compte / moyen enregistré (UUID) |
| `provider_code` | No | défaut `WAVE` — `SPI` possible |
| `payment_method_code` | No | défaut `MOBILE_MONEY` — `BANK_TRANSFER` possible |
| `metadata` | No | Objet clé-valeur (référence facture, etc.) |

## Responses

| Status | Description |
| --- | --- |
| `201` | Paiement initié (statut en attente) |
| `400` | Entrée invalide ou solde insuffisant |
| `401` | Clé API invalide ou manquante |

## Errors

**400** DTO validation, business rules, or insufficient funds. **401** bad key.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/beneficiary-payouts" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000,
    "currency_code": "XOF",
    "provider_code": "WAVE",
    "payment_method_code": "MOBILE_MONEY",
    "metadata": { "reference": "INV-2025-0042" }
  }'
```
