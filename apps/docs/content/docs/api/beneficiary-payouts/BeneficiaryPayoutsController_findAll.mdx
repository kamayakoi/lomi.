## Overview

Paginated list of **beneficiary payouts** for your organization.

- **`statuses`** — comma-separated (e.g. `pending,completed,failed`).
- **`startDate` / `endDate`** — ISO 8601 range.
- **`currencyCode`** — filter (e.g. `XOF`).
- **`limit`** — default **50**; **`offset`** — default **0**.

Response shape: `{ data: BeneficiaryPayoutResponseDto[], limit, offset }`.

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`GET /beneficiary-payouts`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

| Name | Default | Description |
| --- | --- | --- |
| `statuses` | — | Statuts, virgules |
| `startDate` | — | ISO 8601 |
| `endDate` | — | ISO 8601 |
| `currencyCode` | — | ex. `XOF` |
| `limit` | `50` | Max résultats |
| `offset` | `0` | Pagination |

## Responses

| Status | Description |
| --- | --- |
| `200` | Liste paginée des paiements |
| `401` | Clé API invalide ou manquante |

## Errors

**401** missing/invalid key.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/beneficiary-payouts" \
  --data-urlencode "statuses=pending,completed" \
  --data-urlencode "currencyCode=XOF" \
  --data-urlencode "limit=25" \
  --data-urlencode "offset=0" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
