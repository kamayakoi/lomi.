## Overview

Persist a buyer record under your merchant org. **`name`** is mandatory; enrich with email/phone/metadata for invoicing workflows.

Customers created here integrate with subscriptions, invoices, CRM sync, etc., depending on downstream resources you attach later.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)).

## Endpoint

`POST /customers`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

_None._

### Request body

| Field | Required | Notes |
| --- | --- | --- |
| `name` | **Yes** | Display name |
| `email`, `phone_number`, address fields… | Optional | Localization & invoicing helpers |
| `is_business` | Optional | Flag B2B |
| `metadata` | Optional | Merchant-owned JSON |

Minimal example:

```json
{
  "name": "Jane Doe",
  "email": "jane@example.com"
}
```

## Responses

| Status | Description |
| --- | --- |
| `201` | Client créé avec succès |
| `400` | Données d'entrée invalides |
| `401` | Unauthorized |

## Errors

Validation errors (**400**) for bad email formatting, conflicting unique constraints handled by backend.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/customers" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"Jane Doe","email":"jane@example.com"}'
```
