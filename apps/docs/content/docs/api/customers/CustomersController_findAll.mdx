## Overview

Paged customer directory for your organization.

**Filters**

| Query | Meaning |
| --- | --- |
| `search` | Matches name or email |
| `type` | `business` \| `individual` \| `all` |
| `status` | `active` = at least one txn, `inactive` = none, `all` passthrough |
| `page` | 1-based index (default **1**) |
| `pageSize` | Rows per page (default **50**) |

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /customers`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

See Overview table (`search`, `type`, `status`, `page`, `pageSize`).

## Responses

**200** shape:

```json
{
  "customers": [],
  "pagination": {
    "page": 1,
    "pageSize": 50,
    "totalCount": 0,
    "totalPages": 0
  }
}
```

| Status | Description |
| --- | --- |
| `200` | Liste paginée de clients |
| `401` | Clé API invalide ou manquante |

## Errors

**401** bad key.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/customers" \
  --data-urlencode "search=jane" \
  --data-urlencode "status=active" \
  --data-urlencode "page=1" \
  --data-urlencode "pageSize=25" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
