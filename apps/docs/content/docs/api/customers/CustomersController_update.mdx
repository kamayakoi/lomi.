## Overview

Patch fields on an existing customer; **omit untouched fields**.

## Authentication

Merchant API key (`X-API-KEY`). See [Integration overview](/reference/setup/overview).

## Endpoint

`PATCH /customers/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required |
| --- | --- |
| `id` | Customer UUID |

### Query parameters

_None._

### Request body

Subset of create fields (`name`, `email`, contacts, addresses, metadata, etc.)

## Responses

| Status | Description |
| --- | --- |
| `200` | Client mis à jour avec succès |
| `400` | Validation |
| `401` | Unauthorized |
| `404` | Client introuvable ou accès refusé |

## Errors

Same semantics as customer **create** validation plus tenant isolation rules.

## Example

```bash
curl -sS -X PATCH "https://sandbox.api.lomi.africa/customers/YOUR_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"phone_number":"+221771234567"}'
```
