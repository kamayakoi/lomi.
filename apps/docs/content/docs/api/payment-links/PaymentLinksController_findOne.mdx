## Overview

Retrieve configuration and status for a single payment link UUID (URL, activation, product binding, etc.).

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /payment-links/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | Payment link UUID |

### Query parameters

_No query parameters._

## Responses

| Status | Description |
| --- | --- |
| `200` | Détails du lien |
| `401` | Clé API invalide ou manquante |
| `404` | Lien introuvable ou accès refusé |

## Errors

**404** when id is unknown or belongs to another organization.

## Example

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/payment-links/YOUR_LINK_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
