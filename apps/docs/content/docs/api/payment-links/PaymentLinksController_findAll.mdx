## Overview

Paginated listing of **payment links** for your organization. Filter by **link type** (`product` / `instant`) and by **active flag**.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /payment-links`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

| Name | Notes |
| --- | --- |
| `linkType` | `product` \| `instant` |
| `isActive` | String `"true"` or `"false"` (see `PaymentLinksController` — parsed to boolean) |
| `limit` | Default `20` |
| `offset` | Default `0` |

## Responses

**200** — paginated wrapper:

| Field | Meaning |
| --- | --- |
| `data` | `PaymentLinkResponseDto[]` |
| `total` | Total |
| `limit` / `offset` | Echo paging |

| Status | Description |
| --- | --- |
| `200` | Liste paginée de liens de paiement |
| `401` | Clé API invalide ou manquante |

## Errors

**401** if the API key is invalid.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/payment-links" \
  --data-urlencode "linkType=instant" \
  --data-urlencode "isActive=true" \
  --data-urlencode "limit=20" \
  --data-urlencode "offset=0" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
