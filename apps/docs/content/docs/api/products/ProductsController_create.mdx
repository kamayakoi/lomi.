## Overview

Creates a product **with at least one price**. The first price (or the one flagged `is_default`) becomes the fallback when checkout payloads omit `price_id`.

See `CreateProductDto` for full validation (recurring vs one-time, fee models, etc.).

## Authentication

Merchant `X-API-KEY`.

## Endpoint

`POST /products`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Request body

Structured product + `prices[]` array (see `apps/api/src/core/products/dto/create-product.dto.ts`). Minimum: follow server validation (≥1 price).

## Responses

| Status | Description |
| --- | --- |
| `201` | Created |
| `400` | Validation / business rules |
| `401` | Unauthorized |

## Errors

**400** validation failures (missing prices, incompatible cadence, etc.) or business rules. **401** unauthorized.

## Example

Use your real payload from dashboard testing or reference DTO sample in OpenAPI.

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/products" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d @product-payload.json
```
