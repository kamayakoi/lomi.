## Overview

Returns **payment-related transactions** for the authenticated merchant’s organization — created by Lomi during payment flows (not writable via this API).

**Filters:**

- **`status`, `type`, `currency`, `paymentMethod`** — comma-separated lists (e.g. `status=completed,pending`).
- **`startDate` / `endDate`** — ISO 8601 window.
- **`provider`** — provider code filter (e.g. `WAVE`).
- **`isPos`** — optional filter for POS (TPV) transactions (pass as typical boolean query semantics).
- **`page`** (default **1**) and **`pageSize`** (default **50**).

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`GET /transactions`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

| Name | Notes |
| --- | --- |
| `provider` | Code fournisseur (ex. `WAVE`) |
| `status` | Statuts, virgules (ex. `completed,pending`) |
| `type` | Types, virgules (ex. `payment,refund`) |
| `currency` | Devises, virgules (ex. `XOF,USD`) |
| `paymentMethod` | Moyens, virgules (ex. `MOBILE_MONEY,CARDS`) |
| `page` | Page (défaut `1`) |
| `pageSize` | Taille (défaut `50`) |
| `startDate` | ISO 8601 début |
| `endDate` | ISO 8601 fin |
| `isPos` | Filtre TPV / point de vente |

## Responses

**200** — `TransactionResponseDto[]` (paginated per service).

| Status | Description |
| --- | --- |
| `200` | Liste des transactions |
| `401` | Clé API invalide ou manquante |

## Errors

**401** missing/invalid key. **429** if you exceed rate limits — back off and retry.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/transactions" \
  --data-urlencode "provider=WAVE" \
  --data-urlencode "status=completed,pending" \
  --data-urlencode "page=1" \
  --data-urlencode "pageSize=25" \
  --data-urlencode "startDate=2025-01-01T00:00:00Z" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
