## Overview

Initiates an **outbound Wave payout** via the edge integration. The response body mirrors the **Wave edge JSON** (see `PayoutResponseDto` in OpenAPI).

The API **overwrites** `organizationId` and `merchantId` from your authenticated context — you do not need to send them (optional fields on `CreateWavePayoutDto` are for internal use).

**Minimum amount:** **100** (same currency unit as `currency`, typically minor units per DTO).

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`POST /payout/wave`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

_None._

### Request body

JSON `CreateWavePayoutDto`:

| Field | Required | Notes |
| --- | --- | --- |
| `amount` | Yes | `number`, ≥ 100 |
| `currency` | Yes | e.g. `XOF` |
| `beneficiary` | Yes | `{ name, phoneNumber }` (mobile money format e.g. `+221…`) |
| `reason` | No | Motif affiché / traçabilité |

## Responses

| Status | Description |
| --- | --- |
| `201` | Virement initié (corps JSON Wave / edge) |
| `400` | Entrée invalide ou erreur fournisseur |
| `401` | Clé API invalide ou manquante |

## Errors

**400** validation (amount, beneficiary) or Wave/provider error in the edge call. **401** bad key.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/payout/wave" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 5000,
    "currency": "XOF",
    "beneficiary": {
      "name": "Studio Example",
      "phoneNumber": "+221701234567"
    },
    "reason": "Weekly settlement"
  }'
```
