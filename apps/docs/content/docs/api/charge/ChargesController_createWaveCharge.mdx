## Overview

Creates a **direct Wave charge** (collect payment) with a nested **customer** object. The API sets **`organizationId`** and **`merchantId`** from your authenticated context — do not rely on sending those fields for authorization.

Optional fields such as `description`, `successUrl`, `errorUrl`, and `environment` are passed through to the charge flow as supported by the service (see DTO / OpenAPI).

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`POST /charge/wave`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Request body

`CreateWaveChargeDto` (see `apps/api/src/core/charges/dto/create-charge.dto.ts`):

| Field | Required | Notes |
| --- | --- | --- |
| `amount` | Yes | ≥ 1 |
| `currency` | Yes | e.g. `XOF` |
| `customer` | Yes | `{ name, phoneNumber }` — `email` optional but must be valid if set |
| `description` | No | — |
| `successUrl` / `errorUrl` | No | Redirect URLs if applicable |
| `environment` | No | — |

`organizationId` / `merchantId` in the DTO are optional and **overwritten** server-side from the API key.

### Path parameters

_None._

### Query parameters

_None._

## Responses

| Status | Description |
| --- | --- |
| `201` | Encaissement initié (JSON edge Wave) |
| `400` | Entrée invalide ou erreur API Wave |
| `401` | Clé API invalide ou manquante |

## Errors

**400** customer payload, amount/currency, or Wave API error. **401** bad key.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/charge/wave" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": 2500,
    "currency": "XOF",
    "customer": {
      "name": "Awa Diop",
      "phoneNumber": "+221771112233",
      "email": "awa@example.com"
    },
    "description": "Order #1234"
  }'
```
