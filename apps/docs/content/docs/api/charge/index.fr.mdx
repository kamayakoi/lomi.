Les APIs d’encaissement direct sont destinées aux équipes qui ont besoin d’un contrôle plus bas niveau que le checkout hébergé ou les liens de paiement. Pour le flux recommandé par défaut, utilisez les [Checkout Sessions](/api/checkout-sessions).

## Quand les utiliser ?

Utilisez les encaissements directs quand :

- Votre application possède déjà l’interface de paiement.
- Vous devez démarrer un paiement Wave ou MTN MoMo depuis un flux personnalisé.
- Vous devez collecter une carte embarquée avec confirmation côté client.
- Vous savez gérer les états en attente, échec, annulation et complétion.

## Ce qu’il faut prévoir

Une intégration directe doit inclure :

- Une création côté serveur avec clé secrète.
- Des états visibles pour attente et échec.
- Des webhooks pour la réconciliation finale.
- Une gestion sûre des `client_secret` renvoyés pour les flux carte embarqués.
