## Overview

Cancel subscription

Cancels an active subscription. This is the only allowed subscription mutation; billing plan and cycle attributes remain system-managed.


## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`POST /subscriptions/{id}/cancel`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Schema | Description |
| --- | --- | --- | --- |
| `id` | Yes | — | Subscription UUID |

### Query parameters

_No query parameters._


### Request body

JSON request payload.

Schema: `object`

| Field | Required | Type | Description |
| --- | --- | --- | --- |
| `cancellation_reason` | No | `string` | — |

Example body:

```json
{
  "cancellation_reason": "string"
}
```


## Responses

| Status | Description |
| --- | --- |
| `200` | Subscription canceled successfully |
| `401` | API key missing or invalid |
| `404` | Subscription not found or access denied |

## Errors

Errors follow the standard JSON error format (status code and machine-readable message). Validate inputs before calling; **401** indicates a missing/invalid key, **404** a missing resource for this organization, **429** rate limiting. For safe retries on create-style calls, send an idempotency key when your flow supports it.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/subscriptions/value/cancel" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"cancellation_reason":"string"}'
```
