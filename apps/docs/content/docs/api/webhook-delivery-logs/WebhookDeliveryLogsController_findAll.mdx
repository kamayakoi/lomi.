## Overview

Lists **outbound webhook delivery attempts** (success/failure, payload metadata) for a given **`webhookId`**. Logs are created automatically when Lomi tries to notify your endpoint.

**`webhookId` is required** — use **`GET /webhooks`** (registered webhooks listing) or your dashboard to copy the id.

Optional filters: **`successOnly`**, **`failedOnly`** (booleans, default `false`); pagination **`limit`** (default **25**), **`offset`** (default **0**).

For delivery semantics (retries, duplicates), see [Webhook reliability](/core/advanced-guides/webhook-reliability).

## Authentication

Merchant `X-API-KEY` — [Integration overview](/reference/setup/overview).

## Endpoint

`GET /webhook-delivery-logs`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_None._

### Query parameters

| Name | Required | Default | Description |
| --- | --- | --- | --- |
| `webhookId` | **Yes** | — | UUID du webhook |
| `successOnly` | No | `false` | Livraisons réussies uniquement |
| `failedOnly` | No | `false` | Échecs uniquement |
| `limit` | No | `25` | Max entrées |
| `offset` | No | `0` | Pagination |

## Responses

**200** — `WebhookDeliveryLogResponseDto[]`.

| Status | Description |
| --- | --- |
| `200` | Liste des journaux de livraison |
| `401` | Clé API invalide ou manquante |

## Errors

**401** bad key. Omitting **`webhookId`** causes a **400** at runtime (required query param in controller).

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/webhook-delivery-logs" \
  --data-urlencode "webhookId=YOUR_WEBHOOK_UUID" \
  --data-urlencode "failedOnly=true" \
  --data-urlencode "limit=25" \
  --data-urlencode "offset=0" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
