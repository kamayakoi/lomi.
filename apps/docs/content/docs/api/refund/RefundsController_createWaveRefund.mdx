## Overview

Creates a **Wave refund** against an existing **transaction** (`transactionId` UUID). Response shape follows the **edge Wave refund** payload (`RefundResponseDto`).

**Amount** must be ≥ **1** (per `CreateWaveRefundDto` validation).

## Authentication

`X-API-KEY` (**ApiKeyGuard** on controller). Include the same merchant key as for other REST calls.

## Endpoint

`POST /refund/wave`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Request body

| Field | Required | Notes |
| --- | --- | --- |
| `transactionId` | Yes | UUID of the transaction to refund |
| `amount` | Yes | Amount to refund (≥ 1) |
| `reason` | No | Optional reason string |

### Path parameters

_None._

### Query parameters

_None._

## Responses

| Status | Description |
| --- | --- |
| `201` | Résultat du remboursement (JSON edge) |
| `400` | Entrée invalide |
| `401` | Clé API invalide ou manquante |
| `500` | Échec du traitement du remboursement côté service / fournisseur |

## Errors

**400** invalid UUID, amount, or business validation. **500** retry may or may not be safe — record the outcome and reconcile with transactions; see [Webhook reliability](/core/advanced-guides/webhook-reliability) for delivery semantics.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/refund/wave" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "transactionId": "123e4567-e89b-12d3-a456-426614174000",
    "amount": 1000,
    "reason": "Customer request"
  }'
```
