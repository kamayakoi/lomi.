## Overview

Fetch a single checkout session by UUID (status line items, payer metadata populated when applicable).

Only sessions belonging to the **organization bound to your API key** can be fetched.

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /checkout-sessions/{id}`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

| Name | Required | Description |
| --- | --- | --- |
| `id` | Yes | UUID of the checkout session |

### Query parameters

_No query parameters._

## Responses

| Status | Description |
| --- | --- |
| `200` | Détails de la session |
| `401` | Clé API invalide ou manquante |
| `404` | Session introuvable ou accès refusé |

## Errors

**404:** session UUID exists but belongs to another organization → **introuvable** for your key (**do not distinguish** externally).

## Example

Replace `YOUR_SESSION_UUID` with a returned session id:

```bash
curl -sS -X GET "https://sandbox.api.lomi.africa/checkout-sessions/YOUR_SESSION_UUID" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
