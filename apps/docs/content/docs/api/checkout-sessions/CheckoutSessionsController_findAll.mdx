## Overview

List **checkout sessions for the current merchant organization**. Supports pagination and optional filter by lifecycle status (`open`, `completed`, `expired`).

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`GET /checkout-sessions`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

| Name | Defaults | Notes |
| --- | --- | --- |
| `status` | (any) | `open` \| `completed` \| `expired` — filter by checkout session status |
| `limit` | `20` | Max rows per page (`ParseIntPipe`; see `apps/api/src/core/checkout-sessions/checkout-sessions.controller.ts`) |
| `offset` | `0` | Pagination offset |

## Responses

Returns an **array** of `CheckoutSessionResponseDto`.

| Status | Description |
| --- | --- |
| `200` | Liste des sessions de paiement |
| `401` | Clé API invalide ou manquante |

## Errors

**401:** invalid or absent API key **401** Unauthorized.

## Example

```bash
curl -sS -G "https://sandbox.api.lomi.africa/checkout-sessions" \
  --data-urlencode "status=open" \
  --data-urlencode "limit=20" \
  --data-urlencode "offset=0" \
  -H "X-API-KEY: $LOMI_API_KEY"
```
