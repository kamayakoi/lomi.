## Overview

Create a hosted **checkout session**: lomi. returns a session id and checkout URL where the payer completes payment.

**Lifetime:** sessions expire **60 minutes after creation** unless your product logic dictates otherwise elsewhere.

Choose one of three shapes:

| Mode | Typical fields |
| --- | --- |
| **Fixed amount** | `currency_code`, `amount`, URLs |
| **Product / subscription** | `currency_code`, `product_id` and/or `price_id`, optionally `quantity` |
| **Multi-product cart** | `currency_code`, `line_items` (each line needs `price_id`; root `product_id`, `price_id`, `quantity` are ignored — see `CreateCheckoutSessionDto` in `apps/api/src/core/checkout-sessions/dto/create-checkout-session.dto.ts`) |

## Authentication

Merchant routes require an API key in the `X-API-KEY` header (see [Integration overview](/reference/setup/overview)). Use a **test** key against `https://sandbox.api.lomi.africa` and a **live** key against `https://api.lomi.africa`.

## Endpoint

`POST /checkout-sessions`

Base URLs:

- `https://sandbox.api.lomi.africa`
- `https://api.lomi.africa`

## Request

### Path parameters

_No path parameters beyond the URL pattern._

### Query parameters

_No query parameters._

### Request body

`currency_code` is **always required**.

| Field | Required | Role |
| --- | --- | --- |
| `currency_code` | Yes | One of `XOF`, `USD`, `EUR` |
| `amount` | For simple amount checkouts | Smallest currency unit / amount semantics as documented for your merchant account |
| `line_items[]` | For cart checkout | Each item requires `price_id`; when set, root `amount`/`product_id`/`price_id`/`quantity` are ignored (`CheckoutSessionsController`). |
| `customer_*` | Optional | Prefer `customer_id` for known customers |
| `success_url`, `cancel_url` | Recommended | Hosted redirect URLs after payment |
| `payment_link_id` | Optional | Tie session creation to an existing payment link |
| `metadata` | Optional | Your own opaque key/value object |

Example — fixed amount:

```json
{
  "currency_code": "XOF",
  "amount": 10000,
  "success_url": "https://example.com/success",
  "cancel_url": "https://example.com/cancel"
}
```

Example — line items:

```json
{
  "currency_code": "XOF",
  "line_items": [
    { "price_id": "123e4567-e89b-12d3-a456-426614174000", "quantity": 2 }
  ],
  "success_url": "https://example.com/success"
}
```

## Responses

**201** Created — body matches `CheckoutSessionResponseDto` (checkout URL, identifiers, expiration context).

| Status | Description |
| --- | --- |
| `201` | Session créée avec succès |
| `400` | Entrée invalide ou erreur de validation |
| `401` | Clé API invalide ou manquante |

## Errors

Validation errors return **400** with field-level hints when validation fails (`class-validator`). **401** bad or missing `X-API-KEY`. Prefer **retrying creates** behind an idempotent key pattern on your backend if implementing retries.

## Example

```bash
curl -sS -X POST "https://sandbox.api.lomi.africa/checkout-sessions" \
  -H "X-API-KEY: $LOMI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "currency_code":"XOF",
    "amount":10000,
    "success_url":"https://example.com/success",
    "cancel_url":"https://example.com/cancel"
  }'
```
