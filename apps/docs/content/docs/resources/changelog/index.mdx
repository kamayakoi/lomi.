## Version 1

- **Initial release:** The first version of our REST API (no global version prefix in URL paths).
- **Core resources:** Introduced foundational endpoints for managing:
  - [Authentication](/api/authentication)
  - [Providers](/api/data-models#provider-object)
  - [Webhooks](/build/webhooks)
  - [API reference](/api)
  - [Customers](/api/customers)
  - [Plans](/build/subscriptions)
  - [Products](/build/products)
  - [Checkout Sessions](/build/checkout)
  - [Subscriptions](/api/subscriptions)
  - [Payment Links](/build/payment-links)
  - [Transactions](/build/transactions)
  - [Refunds](/build/refunds) (Beta)
- **Environments:** Established separate Test and Live environments.
- **Error handling:** Standardized error response structure.
- **lomi. CLI:** Initial release of the command-line interface `lomi.cli`.
- **Node.js SDK:** Initial release of the `@lomi./sdk` package.
- **Pay what you want:** Hosted checkout supports custom unit amounts within configured bounds. Product prices accept `pricing_model`, `minimum_amount`, and `maximum_amount`; checkout session amount validation respects PWYW rules. See [Products](/build/products) and [Checkout behavior](/build/payments/checkout-behavior).
- **Subscription checkout:** Recurring products support trials (`trial_enabled`), deferred first payment (`non_initial`), prorated first charges, automatic card renewals, mobile-money signup without immediate charge, and manual renewal fallback links. See [Subscriptions](/build/subscriptions) and [Checkout behavior — Subscription checkout](/build/payments/checkout-behavior#subscription-checkout).
- **Subscription webhooks & signup emails:** Merchant `SUBSCRIPTION_CREATED`, `SUBSCRIPTION_RENEWED`, and `SUBSCRIPTION_CANCELLED` webhooks are delivered when subscriptions are created, renewed, or cancelled. Trial and $0 signups send customer signup confirmation emails (configurable via `subscription_signups` notification settings).
