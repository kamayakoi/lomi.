## Version 1

- **Version initiale :** première version de notre API REST (sans préfixe de version global dans les chemins URL).
- **Ressources principales :** endpoints pour notamment :
  - [Authentification](/api/authentication)
  - [Fournisseurs](/api/data-models#provider-object)
  - [Webhooks](/build/webhooks)
  - [Référence API](/api)
  - [Clients](/api/customers)
  - [Plans](/build/subscriptions)
  - [Produits](/build/products)
  - [Sessions de paiement](/build/checkout)
  - [Abonnements](/api/subscriptions)
  - [Liens de paiement](/build/payment-links)
  - [Transactions](/build/transactions)
  - [Remboursements](/build/refunds) (bêta)
- **Environnements :** séparation Test et Live.
- **Gestion d’erreurs :** structure d’erreur normalisée.
- **CLI lomi.** : première version de l’interface `lomi.cli`.
- **SDK Node.js** : première version du paquet `@lomi./sdk`.
- **Payez ce que vous voulez :** le tunnel hébergé accepte des montants unitaires personnalisés dans les bornes configurées. Les tarifs produits acceptent `pricing_model`, `minimum_amount` et `maximum_amount` ; la validation du montant de session respecte les règles PWYW. Voir [Produits](/build/products) et [Comportement du tunnel](/build/payments/checkout-behavior).
- **Abonnements :** produits récurrents avec essais (`trial_enabled`), premier paiement différé (`non_initial`), prorata, renouvellements carte automatiques, inscription mobile money sans prélèvement immédiat et liens de renouvellement manuels. Voir [Abonnements](/build/subscriptions) et [Comportement du tunnel — Abonnements](/build/payments/checkout-behavior).
- **Webhooks abonnement et e-mails d’inscription :** les webhooks marchands `SUBSCRIPTION_CREATED`, `SUBSCRIPTION_RENEWED` et `SUBSCRIPTION_CANCELLED` sont émis à la création, au renouvellement et à l’annulation. Les inscriptions essai ou à 0 € envoient un e-mail de confirmation client (réglable via `subscription_signups`).
